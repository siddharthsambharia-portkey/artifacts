Governance

Control who can deploy, how many documents a site can store, and who can read across site boundaries.

Governance modes

ModeDescriptionBest for
trustOpen — any employee can deploy, update, or delete any site. Cross-site reads are allowed.Small teams, hackathons
governedOwnership enforced. Only the site owner (or admins) can redeploy. Quotas enforced. Cross-site reads still allowed by default.Large companies
governance: mode: "governed" # trust | governed

Ownership in governed mode

In governed mode:

  • The first person to deploy a site name becomes the owner
  • Only the owner (or server admins) can redeploy to that name
  • Anyone can deploy a new site name they don't already own
  • Admins can transfer ownership via the admin console (/admin)
ℹ️

The admin list is configured in artifact.yaml under governance.admins. Admins can redeploy any site and access all audit logs.

Admins

governance: mode: "governed" admins: - "platform-team@corp.com" # group — everyone in this group is an admin - "alice@corp.com" # individual - "bob@corp.com"

Admins are matched against artifact.me.email and artifact.me.groups.

Document quotas

In governed mode, you can limit how many documents a site can store:

governance: mode: "governed" default_doc_quota: 10000 # documents per site (default: unlimited) per_site_quotas: - site: "high-volume-app" doc_quota: 1000000

When a site hits its quota, collection.create() returns a 429 with a descriptive error. The admin can raise the quota from the admin console.

Cross-site reads

By default, any site can read from any other site's collection:

// Read another site's collection const posts = artifact.db.collection('posts', { site: 'team-blog' });

To restrict this in governed mode:

governance: mode: "governed" allow_cross_site_reads: false # default: true in both modes

When disabled, cross-site reads return a 403.

Deploy allowlist

Restrict who can deploy at all:

governance: mode: "governed" deploy_groups: - "employees" # everyone in this SSO group can deploy - "contractors"

If deploy_groups is empty (the default), any authenticated employee can deploy.

Audit log

Every deploy, API write, AI call, and warehouse query is logged. View in the admin console (/admin) or via the CLI:

artifact logs # server-wide audit log artifact logs my-app # single site artifact logs --json | jq . # JSON output for ingestion

Audit records include: timestamp, user email, site, action, resource, HTTP status, and IP address.
