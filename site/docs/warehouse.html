Warehouse queries

Connect BigQuery, Snowflake, or a Postgres read replica and let employees query data from any Artifact site. SELECT-only, dataset allowlist enforced server-side.

Security model

Use read-only credentials. The warehouse login you give Artifact must be read-only at the warehouse itself (a read-only BigQuery role, a Snowflake read-only role, or a Postgres read replica / readonly user). This is the real guarantee that employee queries can never modify your data — the database physically refuses any write, no matter how a query is phrased. Artifact's query checks below are an early, friendly guardrail, not the security boundary.
  • Read-only credentials (the guarantee) — Artifact connects with the read-only login you provide; the warehouse rejects all writes server-side
  • SELECT-only early check — Artifact rejects queries that don't start with SELECT or that contain write keywords (INSERT, UPDATE, DELETE, DROP, …) so Builders get a clear error before the query leaves the building. This is a text check, not a full SQL parser — defense in depth, not the wall.
  • Dataset allowlist — Only tables in listed datasets/schemas can be queried
  • Row limit — Results capped at max_rows (default 10,000)
  • Per-user daily query quota — Rejects when exceeded
  • Credentials server-side — DSN / service account key never reaches the client

BigQuery

warehouse: sources: - name: "bigquery" type: "bigquery" project: "corp-data-prod" credentials_file: "/run/secrets/bq-sa.json" # or use Workload Identity datasets_allowlist: - "analytics" - "hr" - "finance_readonly" max_rows: 10000 daily_query_quota: 1000 # per user

For GKE Workload Identity, omit credentials_file — Artifact will use Application Default Credentials.

Snowflake

warehouse: sources: - name: "snowflake" type: "snowflake" # Snowflake uses a Postgres-compatible DSN via the snowflakedb/gosnowflake driver dsn: "snowflake://user:pass@account.snowflakecomputing.com/CORP_DB" # or set ARTIFACT_WAREHOUSE_SNOWFLAKE_DSN schemas_allowlist: - "ANALYTICS" - "HR" max_rows: 5000

Postgres read replica

warehouse: sources: - name: "analytics" type: "postgres" dsn: "postgres://readonly:...@analytics-replica.internal/corp" # or set ARTIFACT_WAREHOUSE_ANALYTICS_DSN schemas_allowlist: - "public" - "reporting" max_rows: 10000

Multiple sources

You can configure multiple warehouse sources. The client specifies which to use with the source option:

const results = await artifact.warehouse.query(sql, [], { source: 'bigquery' }); const results2 = await artifact.warehouse.query(sql, [], { source: 'snowflake' }); // Without source: uses the first configured source

Parameterized queries

Always use parameterized queries to prevent SQL injection. The driver handles quoting:

const results = await artifact.warehouse.query( 'SELECT * FROM analytics.events WHERE user_id = $1 AND date >= $2', [artifact.me.email, '2026-01-01'] );

Result shape

// results: { columns: ['department', 'headcount', 'avg_tenure'], rows: [ ['Engineering', 142, 2.4], ['Product', 38, 1.9], // ... ], truncated: false, // true if max_rows was hit elapsed_ms: 234 }
