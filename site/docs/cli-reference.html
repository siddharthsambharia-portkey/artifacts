CLI reference

All Artifact CLI commands. Run artifact <command> --help for full flag reference.

Global flags

FlagDefaultDescription
--configartifact.yamlPath to config file
--serverhttp://localhost:8443Server URL for client commands
--tokenAuth token (or set ARTIFACT_TOKEN)

artifact serve

Run the Artifact server in production mode. Reads artifact.yaml for configuration.

artifact serve [flags]
FlagDefaultDescription
--configartifact.yamlConfig file path
--port8443HTTP listen port
--host0.0.0.0HTTP listen address
artifact serve --config /etc/siddharthsambharia-portkey/artifacts.yaml --port 443

artifact dev

Run the dev server with SQLite, local storage, and fake identity. No config required.

artifact dev [flags]
FlagDefaultDescription
--port8443Port to listen on
--data.artifact-dataData directory for SQLite and files

Dev mode automatically:

  • Logs every request to the terminal
  • Creates a dev@localhost session (no SSO)
  • Stores state in .artifact-data/ (gitignored)

artifact init

Scaffold a new Artifact site in the current (or named) directory.

artifact init [site-name]

Creates:

[site-name]/ ├── index.html ← starter page with artifact.js included ├── artifact.json ← { "site": "site-name" } ├── SKILL.md — SDK quick reference ├── AGENTS.md ← SDK skill file for other agents └── README.md
ℹ️

SKILL.md and AGENTS.md tell AI coding assistants the full SDK surface so they can build Artifact apps without reading docs.

artifact deploy

Deploy the current directory (or a named directory) to the Artifact server.

artifact deploy [directory] [flags]
FlagDefaultDescription
--sitefrom artifact.jsonSite name override
--serverhttp://localhost:8443Target server
--excludeGlob patterns to exclude (comma-separated)

Deploy reads the site name from artifact.json in the directory. If there's no artifact.json, it uses the directory name.

cd my-app artifact deploy # deploy . to localhost:8443 artifact deploy --server https://artifact.corp.com artifact deploy --site my-app-v2 # override site name

artifact login

Authenticate with an Artifact server. Opens a browser for OIDC login and saves the token to ~/.config/artifact/token.

artifact login [server-url]
artifact login https://artifact.corp.com # Opens browser → completes OIDC flow → saves token

artifact list

List all sites on the server.

artifact list [flags]
$ artifact list NAME DEPLOY-ID LAST DEPLOY OWNER FILES lunch-poll dep_01J... 2026-06-11 11:08 alice@corp.com 3 team-dashboard dep_01J... 2026-06-11 09:22 alice@corp.com 5 guestbook dep_01J... 2026-06-10 15:41 bob@corp.com 2

artifact open

Open a site in the default browser.

artifact open [site-name]
artifact open lunch-poll # opens http://lunch-poll.localhost:8443

artifact logs

Tail or display the audit log for a site or the whole server.

artifact logs [site-name] [flags]
FlagDefaultDescription
--tail50Number of recent entries to show
--follow, -ffalseStream new entries
--jsonfalseOutput as JSON lines
artifact logs lunch-poll -f --tail 100

artifact mcp

Start the MCP (Model Context Protocol) server so AI coding agents can deploy sites and query data programmatically.

artifact mcp [flags]

Exposes the following MCP tools:

ToolDescription
deploy_siteDeploy a local directory to the Artifact server
list_sitesList all deployed sites
read_logsRead the audit log for a site
query_dbRun a read-only query on a site's document collection

Configure in your AI assistant

Add to your MCP config (e.g. ~/.config/mcp.json):

{ "mcpServers": { "artifact": { "command": "artifact", "args": ["mcp", "--server", "https://artifact.corp.com"] } } }

Now you can prompt: "Build a live voting app and deploy it to Artifact" — and the agent handles everything.
