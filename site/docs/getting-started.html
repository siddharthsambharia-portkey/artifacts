Getting started

Install Artifact, run the dev server, and deploy your first site with a live database and AI in under 5 minutes.

Prerequisites

  • macOS, Linux, or Windows (WSL2)
  • No other dependencies — Artifact ships as a single static binary

Install

The fastest way is the install script:

curl -fsSL https://raw.githubusercontent.com/artifact/artifact/main/scripts/install.sh | sh

This downloads the correct binary for your OS/arch to /usr/local/bin/artifact.

Homebrew (macOS)

brew tap artifact/artifact brew install artifact

Build from source

git clone https://github.com/artifact/artifact cd artifact go build -o bin/artifact ./cmd/artifact export PATH="$PWD/bin:$PATH"
ℹ️

Run artifact version to confirm installation. You should see the version number and build info.

Start the dev server

The dev server runs locally with SQLite, local disk storage, and fake identity (no SSO needed).

artifact dev

Output:

▶ Artifact dev server auth: dev (fake identity) db: .artifact-data/artifact.db storage: .artifact-data/ admin: http://localhost:8443/admin ready: http://<site>.localhost:8443
⚠️

Dev mode logs you in as dev@localhost automatically. Never expose a dev-mode server on a public or shared network.

Create your first site

artifact init my-app cd my-app

This creates a minimal scaffold:

my-app/ ├── index.html ← Your app ├── artifact.json ← Site metadata └── README.md

Deploy

artifact deploy

Output:

Uploading 3 files... ✓ Deployed my-app → http://my-app.localhost:8443
ℹ️

Deploys are atomic. Artifact writes all files then flips a single pointer — your site is never served half-deployed.

Add the SDK

Open index.html and add the SDK script tag:

<!DOCTYPE html> <html> <head><title>My App</title></head> <body> <h1>Hello, <span id="name"></span></h1> <ul id="posts"></ul> <button onclick="addPost()">Add post</button> <script src="/artifact.js"></script> <script> await artifact.ready(); document.getElementById('name').textContent = artifact.me.name; const posts = artifact.db.collection('posts'); // Show existing + new posts in real time posts.subscribe({ onCreate: (doc) => { const li = document.createElement('li'); li.textContent = `${doc.created_by}: ${doc.data.text}`; document.getElementById('posts').appendChild(li); } }); async function addPost() { await posts.create({ text: 'Posted at ' + new Date().toLocaleTimeString() }); } </script> </body> </html>
artifact deploy # redeploy with your changes

Open two browser tabs to http://my-app.localhost:8443. Click "Add post" in one — it appears in the other instantly.

Add AI

In dev mode, the AI proxy needs an upstream URL. In production, the server admin sets the API key. For local testing, you can configure it in artifact.yaml:

ai: enabled: true upstream_url: https://api.openai.com/v1 api_key: sk-... # or set ARTIFACT_AI_API_KEY env var models_allowlist: - gpt-4o-mini
// In your HTML: const stream = await artifact.ai.chat([ { role: 'user', content: 'Summarize today in one sentence' } ], { stream: true }); const out = document.getElementById('output'); for await (const chunk of stream) { out.textContent += chunk; }

Manage sites

artifact list # show all sites on this server artifact open my-app # open site in browser artifact logs my-app # tail audit log for site

Admin console

Visit http://localhost:8443/admin while the dev server is running. You'll see:

  • All deployed sites and their deploy history
  • AI usage by user
  • Audit log
  • Server config and quotas
