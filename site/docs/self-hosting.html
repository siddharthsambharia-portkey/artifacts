Self-hosting

Deploy Artifact at your company in 30 minutes. One binary, pluggable auth, storage, and database. Ships with Docker Compose, Helm, and Terraform examples.

artifact.yaml

All configuration is in a single artifact.yaml file. Environment variables override any field using the pattern ARTIFACT_SECTION_KEY.

server: host: "0.0.0.0" port: 8443 base_domain: "artifact.corp.com" # wildcard DNS: *.artifact.corp.com auth: mode: "oidc" # dev | oidc | header-trust oidc: issuer: "https://corp.okta.com" client_id: "0oa..." client_secret: "" # use ARTIFACT_AUTH_OIDC_CLIENT_SECRET redirect_url: "https://artifact.corp.com/auth/callback" groups_claim: "groups" db: type: "postgres" # sqlite | postgres url: "" # use ARTIFACT_DB_URL # For SQLite: path: ".artifact-data/artifact.db" storage: type: "s3" # local | s3 | gcs s3: bucket: "artifact-prod" region: "us-east-1" endpoint: "" # leave empty for AWS; set for MinIO access_key: "" # use ARTIFACT_STORAGE_S3_ACCESS_KEY secret_key: "" # use ARTIFACT_STORAGE_S3_SECRET_KEY governance: mode: "governed" # trust | governed ai: enabled: true upstream_url: "https://api.openai.com/v1" api_key: "" # use ARTIFACT_AI_API_KEY models_allowlist: - "gpt-4o-mini" - "gpt-4o" daily_token_quota: 100000 # per user, per day notify: slack: webhook_url: "" # use ARTIFACT_NOTIFY_SLACK_WEBHOOK_URL channels_allowlist: - "#general" - "#eng-alerts"

Option 1 — Single VM

The simplest production deployment: one VM, Postgres, S3.

# On the VM: curl -fsSL https://raw.githubusercontent.com/siddharthsambharia-portkey/artifacts/main/scripts/install.sh | sh # Write config cat > /etc/siddharthsambharia-portkey/artifacts.yaml <<'EOF' server: base_domain: artifact.corp.com auth: mode: oidc oidc: issuer: https://corp.okta.com client_id: 0oa... redirect_url: https://artifact.corp.com/auth/callback db: type: postgres storage: type: s3 s3: bucket: artifact-prod region: us-east-1 EOF # Set secrets as env vars (or use a secrets manager) export ARTIFACT_AUTH_OIDC_CLIENT_SECRET=... export ARTIFACT_DB_URL=postgres://artifact:...@db.internal/artifact export ARTIFACT_STORAGE_S3_ACCESS_KEY=... export ARTIFACT_STORAGE_S3_SECRET_KEY=... export ARTIFACT_AI_API_KEY=sk-... artifact serve --config /etc/siddharthsambharia-portkey/artifacts.yaml

Option 2 — Docker Compose

The repo ships a deploy/docker-compose.yml that starts Artifact + Postgres + MinIO in one command.

git clone https://github.com/siddharthsambharia-portkey/artifacts cd artifact/deploy # Edit docker-compose.yml or set env vars: export ARTIFACT_AUTH_OIDC_CLIENT_ID=... export ARTIFACT_AUTH_OIDC_CLIENT_SECRET=... export ARTIFACT_AUTH_OIDC_ISSUER=https://corp.okta.com export ARTIFACT_AI_API_KEY=sk-... docker compose up -d

Artifact will be available on port 8443 of your VM. Put it behind an internal load balancer with TLS termination.

Option 3 — Kubernetes (Helm)

helm repo add artifact https://artifact.github.io/helm-charts helm install artifact siddharthsambharia-portkey/artifacts \ --namespace artifact --create-namespace \ -f my-values.yaml

Minimal my-values.yaml:

server: baseDomain: artifact.corp.com auth: mode: oidc oidc: issuer: https://corp.okta.com clientId: 0oa... redirectUrl: https://artifact.corp.com/auth/callback existingSecret: artifact-oidc-secret # key: clientSecret db: type: postgres existingSecret: artifact-db-secret # key: url storage: type: s3 s3: bucket: artifact-prod region: us-east-1 existingSecret: artifact-s3-secret # keys: accessKey, secretKey replicaCount: 3 nats: enabled: true # required for multi-replica WS events

DNS setup

Add a wildcard DNS record pointing to your internal load balancer:

*.artifact.corp.com CNAME internal-lb.corp.com artifact.corp.com CNAME internal-lb.corp.com

With this, my-app.artifact.corp.com routes to Artifact which extracts my-app as the site name.

TLS

Artifact itself does not terminate TLS (it's a Go HTTP server). Put it behind:

  • Nginx / Caddy as a TLS-terminating reverse proxy
  • AWS ALB with ACM wildcard cert
  • GCP Internal Load Balancer with a managed cert
  • cert-manager in Kubernetes with a wildcard Let's Encrypt cert

Health checks

GET /health → 200 OK {"status":"ok","version":"v0.1.0"}

Migrations

Database migrations run automatically on startup. They are embedded in the binary and idempotent — safe to run multiple times. No manual migration step required.

Upgrading

# Single VM: curl -fsSL https://raw.githubusercontent.com/siddharthsambharia-portkey/artifacts/main/scripts/install.sh | sh systemctl restart artifact # Docker: docker compose pull && docker compose up -d # Helm: helm upgrade artifact siddharthsambharia-portkey/artifacts -f my-values.yaml
ℹ️

Artifact will warn if the running server version differs from the CLI version. Always upgrade CLI and server together.
