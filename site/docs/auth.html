Auth setup

Configure Artifact to authenticate employees via your SSO provider. Three modes supported: OIDC, header-trust, and dev.

Auth modes

ModeUse case
devLocal development. Fake identity, no SSO.
oidcDirect OIDC to Okta, Entra, Google Workspace.
header-trustTrust identity headers from IAP, Pomerium, Cloudflare Access, etc.

Okta

1. Create an Okta application

  1. Okta Admin Console → Applications → Create App Integration
  2. Sign-in method: OIDC – Web Application
  3. Application name: Artifact
  4. Sign-in redirect URI: https://artifact.corp.com/auth/callback
  5. Sign-out redirect URI: https://artifact.corp.com
  6. Controlled access: Allow everyone (or your Artifact group)
  7. Save and note the Client ID and Client secret

2. Enable groups claim

  1. Okta Admin → Security → API → Authorization Servers → default
  2. Claims tab → Add Claim
  3. Name: groups, Include in token: ID Token, Value type: Groups
  4. Filter: Starts with → (empty) to include all groups

3. Configure artifact.yaml

auth: mode: oidc oidc: issuer: "https://corp.okta.com" # your Okta domain client_id: "0oaXXXXXXXXXXXXXX" client_secret: "" # set ARTIFACT_AUTH_OIDC_CLIENT_SECRET redirect_url: "https://artifact.corp.com/auth/callback" groups_claim: "groups"
⚠️

Never put client_secret in the YAML file checked into git. Use the environment variable ARTIFACT_AUTH_OIDC_CLIENT_SECRET instead.

Microsoft Entra (Azure AD)

1. Register the application

  1. Azure Portal → Microsoft Entra ID → App registrations → New registration
  2. Name: Artifact
  3. Redirect URI: https://artifact.corp.com/auth/callback
  4. Note the Application (client) ID and Directory (tenant) ID
  5. Certificates & secrets → New client secret

2. Add groups claim

  1. Token configuration → Add groups claim → Security groups (or All groups)
  2. In the ID token section, ensure "groups" claim is included

3. Configure artifact.yaml

auth: mode: oidc oidc: issuer: "https://login.microsoftonline.com/{tenant-id}/v2.0" client_id: "{application-client-id}" client_secret: "" # set ARTIFACT_AUTH_OIDC_CLIENT_SECRET redirect_url: "https://artifact.corp.com/auth/callback" groups_claim: "groups" # Entra sends group Object IDs; map in governance config if needed

Google Workspace

1. Create an OAuth 2.0 client

  1. Google Cloud Console → APIs & Services → Credentials → Create Credentials → OAuth client ID
  2. Application type: Web application
  3. Authorized redirect URI: https://artifact.corp.com/auth/callback
  4. Note the Client ID and Client secret

2. Configure artifact.yaml

auth: mode: oidc oidc: issuer: "https://accounts.google.com" client_id: "{client-id}.apps.googleusercontent.com" client_secret: "" # set ARTIFACT_AUTH_OIDC_CLIENT_SECRET redirect_url: "https://artifact.corp.com/auth/callback" groups_claim: "hd" # Google sends hosted domain; use groups_claim: "" and manage groups via governance
ℹ️

Google Workspace OIDC does not include directory groups by default. Consider using header-trust with Google IAP if you need group-based governance.

Header-trust mode (IAP / Pomerium / Cloudflare Access)

If you front Artifact with an identity-aware proxy (Google IAP, Pomerium, Cloudflare Access, oauth2-proxy), you can delegate all authentication to the proxy and have it forward identity headers.

auth: mode: header-trust header_trust: email_header: "X-Forwarded-Email" # header containing user email name_header: "X-Forwarded-User" # optional: user display name groups_header: "X-Forwarded-Groups" # optional: comma-separated groups
🚨

Only use header-trust when Artifact is unreachable except through the proxy. If a user can hit Artifact directly (bypassing the proxy), they can forge identity headers. Bind Artifact to 127.0.0.1 or a private subnet, and only allow traffic from your proxy.

Google IAP example

auth: mode: header-trust header_trust: email_header: "X-Goog-Authenticated-User-Email" # Google IAP prefixes values with "accounts.google.com:" # Artifact strips this prefix automatically

Pomerium example

auth: mode: header-trust header_trust: email_header: "X-Pomerium-Claim-Email" name_header: "X-Pomerium-Claim-Name" groups_header: "X-Pomerium-Claim-Groups"

Session lifetime

auth: session_ttl: "12h" # how long sessions last (default: 12h)
