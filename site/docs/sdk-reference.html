SDK reference

The Artifact browser SDK is a single JavaScript file served at /artifact.js by every Artifact server. It exposes all platform primitives — database, AI, files, WebSockets, and more — via a zero-config API.

Loading the SDK

<script src="/artifact.js"></script>

That's it. No npm, no bundler, no environment variables. The SDK is automatically available as the global artifact object.

artifact.ready()

Call await artifact.ready() before accessing artifact.me or making API calls. It fetches the current user session from the server.

await artifact.ready(); console.log(artifact.me); // now populated
ℹ️

Top-level await works in modern browsers when your script tag uses type="module". For classic scripts, wrap in an async IIFE.

Identity — artifact.me

After await artifact.ready(), artifact.me contains the verified identity of the current employee. This comes from your SSO provider (OIDC) or trusted proxy header — it cannot be spoofed by the client.

FieldTypeExample
artifact.me.emailstring"alice@corp.com"
artifact.me.namestring"Alice Chen"
artifact.me.groupsstring[]["employees","eng"]
artifact.me.substringIdP subject identifier
ℹ️

In dev mode, artifact.me is { email: "dev@localhost", name: "Dev User", groups: ["employees"] }.

Database

Artifact provides document collections backed by Postgres or SQLite. Each document is automatically stamped with the site, creator, and timestamp.

artifact.db.collection(name, options?)

Returns a Collection object scoped to the current site (or another site in trust mode).

const posts = artifact.db.collection('posts'); // Cross-site read (trust mode only) const shared = artifact.db.collection('posts', { site: 'team-blog' });

collection.create(data)

const doc = await posts.create({ title: 'Hello', body: 'World' }); // Returns: // { // id: "01J...", // site: "my-app", // collection: "posts", // created_by: "alice@corp.com", ← server-stamped, cannot be forged // created_at: "2026-06-11T...", // data: { title: "Hello", body: "World" } // }

collection.list(options?)

const docs = await posts.list({ order: '-created_at', // prefix with - for descending limit: 50, // max 1000 offset: 0, });

collection.update(id, data)

await posts.update(doc.id, { title: 'Updated title' }); // Merges with existing data (partial update)

collection.delete(id)

await posts.delete(doc.id);

Realtime — subscribe

Subscribe to live document changes over WebSocket. Callbacks fire for every create, update, and delete in the collection — across all users and sessions.

collection.subscribe(handlers)

const off = posts.subscribe({ onCreate: (doc) => { console.log('New post:', doc.data.title, 'by', doc.created_by); }, onUpdate: (doc) => { console.log('Updated:', doc.id); }, onDelete: (doc) => { console.log('Deleted:', doc.id); } }); // Unsubscribe off();
ℹ️

The SDK maintains a single WebSocket per room and automatically reconnects on disconnect. In multi-replica deployments, NATS ensures events reach all replicas.

Key-value store

Lightweight, per-site key-value storage. Useful for preferences, feature flags, and per-user state.

// Set a value (any JSON-serializable value) await artifact.kv.set('theme', 'dark'); await artifact.kv.set('onboarding', { step: 3, completed: false }); // Get a value const theme = await artifact.kv.get('theme'); // → "dark" // Scoped to this site; different sites have independent namespaces

artifact.kv.set(key, value)

ParamTypeNotes
keystringMax 256 chars
valueany JSONStored as JSON; max 64 KB

AI proxy

Proxy for LLM chat completions and image generation. The API key lives on the server — never visible in client code. Usage is attributed to the calling user.

⚠️

AI must be explicitly enabled by the server admin. If it's not configured, these calls return a 503.

artifact.ai.chat(messages, options?)

// Non-streaming const res = await artifact.ai.chat([ { role: 'system', content: 'You are a helpful assistant.' }, { role: 'user', content: 'Summarize these bullet points: ...' } ]); console.log(res.choices[0].message.content); // Streaming (async iterable) const stream = await artifact.ai.chat(messages, { stream: true }); for await (const chunk of stream) { output.textContent += chunk; // chunk is a string delta } // With model override (must be in server allowlist) const res = await artifact.ai.chat(messages, { model: 'gpt-4o' });

artifact.ai.image(prompt, options?)

const result = await artifact.ai.image('a watercolor fox at sunset', { size: '1024x1024', // default n: 1, // number of images }); // result.url — a served URL (stored in your object storage) img.src = result.url;

Files

Upload files from the browser. Files are stored in your configured object store (S3, GCS, MinIO, or local). Executable content types are blocked server-side.

artifact.files.upload(file)

const input = document.getElementById('file-input'); input.addEventListener('change', async (e) => { const file = e.target.files[0]; const result = await artifact.files.upload(file); // result: { id, name, size, content_type, url, created_by, created_at } console.log('Uploaded:', result.url); img.src = result.url; // immediately serveable });

artifact.files.list()

const files = await artifact.files.list(); // Returns array of { id, name, size, content_type, url, created_by, created_at } files.forEach(f => console.log(f.name, f.url));

WebSockets

Named rooms with identity-stamped messages. Every message includes the sender's verified identity — clients cannot impersonate others.

artifact.ws.room(name)

const room = artifact.ws.room('all-hands'); // Listen for messages room.on('message', (msg) => { console.log(msg.from.name, '→', msg.payload); // msg.from contains the server-stamped sender identity }); // Send to everyone in the room (including yourself) room.send({ action: 'raise-hand' }); // Leave and disconnect room.close();

Presence

// Subscribe to who's in the room room.presence.subscribe((users) => { header.textContent = `${users.length} online`; // users: [{ email, name, ... }, ...] });

Warehouse queries

Execute read-only SQL against your company's data warehouse. SELECT-only (server-enforced), dataset allowlist, and per-user row limits.

⚠️

Warehouse must be configured by the server admin with a DSN and dataset allowlist. See Warehouse setup.

// Simple query const results = await artifact.warehouse.query( 'SELECT department, COUNT(*) as headcount FROM hr.employees GROUP BY 1 ORDER BY 2 DESC' ); // results: { rows: [...], columns: ['department', 'headcount'] } // With parameters (prevents SQL injection) const results = await artifact.warehouse.query( 'SELECT * FROM sales.orders WHERE region = $1 LIMIT 100', ['EMEA'] ); // Named source (if multiple warehouses configured) const results = await artifact.warehouse.query(sql, [], { source: 'bigquery' });

Notify — Slack

Post messages to Slack channels. The webhook URL and bot token live on the server; the client only specifies the channel and message text.

⚠️

Channel must be in the server's channels_allowlist. Attempts to post to unlisted channels return a 403.

await artifact.notify.slack('#approvals', `${artifact.me.name} approved request #${id}`); // With blocks (Slack Block Kit) await artifact.notify.slack('#eng-alerts', { text: 'Deploy started', blocks: [ { type: 'section', text: { type: 'mrkdwn', text: `*Deploy:* ${siteName}` } } ] });
