AI gateway

The Artifact AI gateway proxies LLM and image generation calls from client code. API keys never leave the server. Every call is attributed to the calling user and counted against their quota.

How it works

When a site calls artifact.ai.chat(), the SDK makes a POST to /api/v1/ai/chat on the Artifact server. The server:

  1. Checks the user's daily token quota — rejects if exceeded
  2. Validates the requested model is in the allowlist
  3. Forwards the request to the upstream LLM API, injecting the server-held API key
  4. Streams the response back to the client
  5. Records usage (input tokens, output tokens, model, user, site) in the ai_usage table

Configuration

ai: enabled: true # Where to send requests. Must be OpenAI-compatible. upstream_url: "https://api.openai.com/v1" # API key — always use env var, never hardcode in YAML api_key: "" # set ARTIFACT_AI_API_KEY # Only these models can be requested by client code. # If empty, all models are allowed (not recommended in production). models_allowlist: - "gpt-4o-mini" - "gpt-4o" - "gpt-image-1" # for image generation # Per-user daily token limit (input + output combined) daily_token_quota: 100000 # 0 = unlimited

OpenAI

ai: enabled: true upstream_url: "https://api.openai.com/v1" models_allowlist: ["gpt-4o-mini", "gpt-4o", "gpt-image-1"]
export ARTIFACT_AI_API_KEY=sk-proj-...

Azure OpenAI

ai: enabled: true # Azure OpenAI endpoint: https://{resource}.openai.azure.com/openai/deployments/{deployment} upstream_url: "https://corp-openai.openai.azure.com/openai/deployments/gpt-4o/v1" models_allowlist: ["gpt-4o"]
export ARTIFACT_AI_API_KEY={your-azure-api-key}

Anthropic (via proxy)

Anthropic's API is not OpenAI-compatible. Use a translation proxy such as LiteLLM in front:

# docker-compose addition: litellm: image: ghcr.io/berriai/litellm:main-latest environment: ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY} command: --config /config.yaml --port 8080
ai: enabled: true upstream_url: "http://litellm:8080/v1" # points to LiteLLM api_key: "litellm-master-key" models_allowlist: ["claude-3-5-sonnet-20241022", "claude-opus-4-8"]

SSRF protection

Artifact validates the upstream_url on startup and blocks requests to private IP ranges, localhost, and link-local addresses. This prevents a compromised site from using the AI proxy to reach internal services.

Quotas and monitoring

Daily token quotas reset at midnight UTC. View usage in the admin console (/admin) or query the API:

GET /api/v1/admin/ai-usage?days=7 → [{ user, site, model, input_tokens, output_tokens, day }, ...]
⚠️

When a user hits their quota, artifact.ai.chat() rejects with a 429. Catch this in your site code and show a friendly message.

Image generation

Image generation requires a model that supports it (e.g. gpt-image-1 or DALL-E). Generated images are uploaded to your configured object storage and returned as a served URL — the client never gets a temporary OpenAI URL.

ai: models_allowlist: - "gpt-4o-mini" - "gpt-image-1" # required for artifact.ai.image()
