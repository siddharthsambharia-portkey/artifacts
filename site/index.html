Shopify built an internal platform where anyone drops HTML and gets a live URL with database and AI baked in — 50,000+ sites on a single $200/month VM, used by over half the company. They can't open-source it because it's welded to their GCP + IAP stack. Artifact is the same idea, rebuilt on pluggable parts so any company can run it behind their own Okta, on their own cloud. Platform features

Everything an internal app needs,
zero configuration

Every primitive lives on the server. Client code never touches API keys, auth tokens, or database credentials.

🗄️

Realtime database

Firebase-style document collections. Create, read, update, delete, and subscribe to live changes over WebSocket.

artifact.db.collection('posts').subscribe(…)
🪪

Employee identity

Every request is a verified employee. artifact.me gives you name, email, team, Slack handle, and SSO groups — no auth code ever.

artifact.me.email / .name / .team
🤖

AI proxy

Streaming LLM calls and image generation. API keys live on the server; per-user cost attribution and quotas are automatic.

artifact.ai.chat([…], { stream: true })
📁

File uploads

Upload files from the browser; get back a served URL. Stored in S3/GCS/MinIO. Executable types blocked at the server.

artifact.files.upload(file) → { url }
⚡

WebSockets & presence

Named rooms, identity-attached presence, multiplayer. Build live polls, collaborative cursors, and games in one line.

artifact.ws.room('lobby').send(…)
🔍

Warehouse queries

Read-only SQL against BigQuery, Snowflake, or Postgres replicas. SELECT-only parser, dataset allowlist, row limit.

artifact.warehouse.query('SELECT …')
🔔

Slack notifications

Post to Slack channels from client code. Webhook URL lives on the server; channel allowlist enforced.

artifact.notify.slack('#ops', 'Approved!')
🔑

Key-value store

Lightweight per-site key-value storage for preferences, state, and config. Simple get/set API.

artifact.kv.set('theme', 'dark')
🤝

Cross-site reads

In trust mode, read another site's collection in one line. Build shared dashboards and data ecosystems.

db.collection('posts', { site: 'blog' })
SDK examples

One script tag. Every primitive.

Drop <script src="/artifact.js"></script> into any HTML file deployed on Artifact.

// Firebase-style documents, server stamps site + user const posts = artifact.db.collection('posts'); // Create const doc = await posts.create({ title: 'Hello', body: 'World' }); // → { id, site, created_by, created_at, data: { title, body } } // List with ordering const all = await posts.list({ order: '-created_at', limit: 50 }); // Update & delete await posts.update(doc.id, { title: 'Updated' }); await posts.delete(doc.id); // Cross-site read (trust mode) const shared = artifact.db.collection('posts', { site: 'team-blog' });
// Subscribe to realtime changes over WebSocket const votes = artifact.db.collection('votes'); // Subscribe — auto-reconnects, replays on reconnect const off = votes.subscribe({ onCreate: (doc) => renderVote(doc), onUpdate: (doc) => updateBar(doc), onDelete: (doc) => removeItem(doc.id), }); // Create from another user — every subscriber fires immediately await votes.create({ option: 'Pizza', voter: artifact.me.email }); // Unsubscribe off();
// Keys live on the server — never in client code // Streaming chat const stream = await artifact.ai.chat([ { role: 'user', content: 'Summarize this week in one paragraph' } ], { stream: true }); for await (const chunk of stream) { output.textContent += chunk; } // Image generation const img = await artifact.ai.image('a watercolor fox at sunset'); document.getElementById('preview').src = img.url; // → Served URL, stored on your object storage
// Named rooms with identity-attached presence const room = artifact.ws.room('all-hands'); // Receive messages from anyone in the room room.on('message', (msg) => { console.log(msg.from.name, msg.payload); // identity is server-stamped }); // Send to everyone in the room room.send({ action: 'vote', choice: 'yes' }); // Presence — who's online right now room.presence.subscribe((users) => { console.log(`${users.length} people online`); });
// Identity flows from your SSO — no auth code needed await artifact.ready(); console.log(artifact.me); // { // email: "alice@corp.com", // name: "Alice Chen", // title: "Senior Engineer", // team: "Platform", // slack: "@alice", // groups: ["employees", "platform-team", "on-call"] // } // Server always stamps documents with creator identity // — clients can never forge who created something const doc = await artifact.db.collection('feedback').create({ text: 'Great demo!' }); console.log(doc.created_by); // "alice@corp.com" — server-stamped
Architecture

One binary, wildcard DNS, done

No microservices. No orchestration. A single Go binary runs the entire product on a 2 vCPU VM.

1

Point wildcard DNS

Add *.artifact.corp.com → internal-lb to your DNS. One entry serves all sites.

2

Authenticate with your IdP

Artifact speaks OIDC directly to Okta, Entra, or Google Workspace — or trusts headers from your IAP/Pomerium proxy.

3

Deploy a folder of HTML

artifact deploy uploads files → writes manifest → flips atomic pointer. Never serves half a site.

4

SDK auto-wires

Every request carries a verified session cookie. The SDK makes same-origin calls — no keys, no auth code, no CORS.

Runtime

FAQ

Common questions

Why no custom backends or cron? +
Custom backends mean per-site servers, on-call ownership, and a security review for every throwaway tool. Artifact's bet: a fixed set of primitives covers 95% of internal apps when you're already inside the trust bubble. When something feels missing, the answer is almost always "combine two SDK primitives."
How is this different from Shopify's Quick? +
Quick is welded to Shopify's GCP + IAP stack. Artifact is the same idea rebuilt on pluggable parts — Okta or Entra or Google OIDC, S3 or GCS or MinIO, Postgres or SQLite — Apache-2.0, self-hostable by any company.
Can I expose a site to the public internet? +
No, and this is intentional. Artifact is designed for internal-only deployment behind corporate SSO. The trust bubble assumption is what makes the primitives safe and simple. Never put Artifact on a public ingress.
How does it scale? +
A single 2 vCPU / 4 GB VM comfortably serves a 5,000-person company for static hosting and API calls. For multi-replica: external Postgres + NATS pubsub + sticky sessions, all documented in the Helm chart values.
How do coding agents use this? +
artifact init drops a CLAUDE.md / AGENTS.md skill file so Claude Code and other agents know the full SDK without reading docs. artifact mcp exposes deploy, list, logs, and query_db as MCP tools — so "build a live voting app and ship it" is a single prompt.
